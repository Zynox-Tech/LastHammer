# Last Hammer — Expenditure Management System

A comprehensive desktop-based expenditure, operational tracking, and business management system developed for **Last Hammer Mining Co.**

The application is designed to centralize company expenditures, equipment operations, fuel records, truck activities, royalty payments, financial ledgers, production records, invoices, and business reporting within a structured desktop environment.

Built using **Python, PyQt5, and SQLite**, the system provides a professional offline solution for managing day-to-day mining company operations and financial records.

# Project Overview

The **Last Hammer Expenditure Management System** is a desktop business management application developed to organize and digitize the financial and operational activities of Last Hammer Mining Co.

The system replaces scattered manual records with a centralized digital platform for recording, managing, searching, and reviewing company operations.

The application supports multiple operational areas including:

* General Expenditures
* Royalty Payments
* Excavator Operations
* Dumper and Dumprei Records
* Truck Expenditures
* Fuel Tracking
* Advances
* Land Rent
* Production Records
* Debit and Credit Ledger
* People Register
* Memory Notes
* Invoice Management
* Business Intelligence
* Daily Reports

The application follows a modular desktop architecture:

```text
PyQt5 User Interface
        ↓
Dashboard and Module Interfaces
        ↓
Business and Operational Modules
        ↓
Reporting and Invoice Services
        ↓
SQLite Database
```

# Features

## General Expenditure Management

The general expenditure module provides functionality for recording and managing routine company expenses.

The module supports:

* Expense date management
* Expenditure descriptions
* Expense amount tracking
* Structured expenditure records
* Historical expense storage

This provides a centralized system for maintaining day-to-day company expenditure records.

## Royalty Payment Management

The royalty module is designed to manage royalty-related financial records.

The system supports:

* Royalty payment dates
* Payment details
* Truck quantity records
* Royalty amount tracking
* Historical royalty records

This allows royalty expenditures to be managed separately from general company expenses.

## Excavator Expenditure Management

# Application Architecture

The project follows a modular Python desktop application architecture.

```text
Presentation Layer
        ↓
PyQt5 Dashboard and UI Components
        ↓
Operational Modules
        ↓
Reports and Business Services
        ↓
Database Access Layer
        ↓
SQLite Database
```

The architecture separates user interface components, operational modules, reporting functionality, configuration, and database management.

This modular structure improves maintainability and allows individual business modules to be expanded independently.

# Installation and Setup

## Requirements

Before running the application, install:

* Python
* pip
* Git
* PyQt5

Verify the Python installation:

```bash
python --version
```

Verify pip:

```bash
pip --version
```

---

# Clone the Repository

Clone the project using Git:

```bash
git clone https://github.com/Zynox-Tech/LastHammer.git
```

Navigate to the project directory:

```bash
cd LastHammer
```

---

# Install Dependencies

Install PyQt5:

```bash
pip install PyQt5
```

Additional dependencies required by individual reporting or email components should be installed according to the Python imports used by those modules.

---

# Run the Application

Start the desktop application using:

```bash
python main.py
```

When the application starts, the system initializes the required database tables, upgrades supported database structures, initializes additional modules, performs the database backup process, and launches the main dashboard.

# Database Architecture

The application uses SQLite for local data storage.

The database architecture includes records for:

* General Expenditures
* Royalty Payments
* Excavator Expenditure
* Dumprei Expenditure
* Truck Expenditure
* Memory Notes
* Debit and Credit Ledger
* Application Settings

Additional database initialization functionality supports:

* Fuel Tracking
* Land Rent
* People Register
* Additional Operational Modules

The application uses SQLite Write-Ahead Logging for database operations.

# Local Data Management


The local SQLite database stores operational and financial records.
